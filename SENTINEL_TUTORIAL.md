# Alibaba Sentinel 系统学习教程

欢迎来到 Sentinel 的学习之旅！Sentinel 是阿里巴巴开源的面向分布式、多语言集群的**流量控制**和**熔断降级**组件，被誉为微服务高可用的“护城河”。

---

## 🚀 一、 为什么需要 Sentinel？

在分布式架构中，服务之间的依赖错综复杂。如果某个底层服务出现故障，或者突然遭遇超出系统承载能力的巨大流量，可能会导致整个系统的雪崩（Cascading Failure）。
Sentinel 的主要目标就是解决这些问题：
- **流量控制（限流）**：把流量限制在系统所能承受的范围内，保护系统不被冲垮。
- **熔断降级**：当依赖的下游服务不稳定时，暂时切断对该服务的调用，快速返回失败，防止级联故障。
- **系统负载保护**：从系统的整体维度（如 CPU 使用率、Load 等）进行自适应保护。

---

## 🧠 二、 Sentinel 的核心概念

学习 Sentinel，必须掌握以下两个核心概念：

### 1. 资源 (Resource)
**资源是 Sentinel 的核心测试点**。它可以是任何东西：一段代码、一个函数、一个 HTTP 接口、一次数据库响应。只要配置了 Sentinel 的 `@SentinelResource` 注解，或者通过 API 定义的，都可以称为资源。

### 2. 规则 (Rule)
围绕资源设置的各种限制条件就是规则。所有的防护机制，都是基于作用在资源上的规则来生效的。Sentinel 包含了**五大核心规则**。

---

## 🛡️ 三、 五大核心规则详解

### 1. 流控规则 (Flow Control)
监控应用流量的 QPS 或并发线程数等指标，当达到指定的阈值时对流量进行控制，以避免被瞬时的流量高峰冲垮。
* **基于 QPS**：一秒内的请求数超过设定值，开始限流（例如：1秒内最多处理100条请求）。
* **基于并发线程数**：同时处理该请求的线程数超过阈值，开始限流（保护业务线程池不被耗尽）。
* **流控效果**：
    - **快速失败（直接拒绝）**：默认方式，超出直接抛出 `FlowException`。
    - **Warm Up（预热）**：让系统的通过量缓慢增加，在一定时间内逐渐达到系统 QPS 阈值，避免系统冷启动时被大流量压垮。
    - **排队等待（匀速排队）**：让请求匀速通过（使用漏桶算法），可以应对突发流量。

### 2. 熔断降级规则 (Circuit Breaking / Downgrade)
当检测到调用链路中某个资源出现不稳定运行状态（例如：响应时间长或抛出异常多）时，Sentinel 会对该资源的调用进行干预（熔断），让请求快速失败。支持三种策略：
* **慢调用比例**：请求的响应时间一直很慢，且慢调用的比例超出阈值。
* **异常比例**：请求不断抛出异常，且异常所占的比例超出阈值（如错误率超过 50%）。
* **异常数**：一分钟内请求抛出异常的绝对数量超过了设定的阈值。

### 3. 热点参数限流规则 (Hot Param Flow)
对于特定的、非常频繁访问的参数进行限流。比如一个商品查询接口（`/sku/{id}`），你只想对 ID 为 `iphone15` 的热点请求进行限流，而不影响其他商品的查询。

### 4. 系统保护规则 (System Protection)
这是一种**全局保护**手段。结合系统的 Load、CPU 使用率、入口 QPS、平均响应时间等整体指标进行自适应防护，防止整体系统雪崩。这种保护是针对整个应用实例的提升，无关乎单一资源。

### 5. 授权规则 (Authority / Black-White List)
控制哪些上游调用方允许访问，哪些不允许。例如，只允许来自 `gateway` 或特定服务 IP 的请求访问内部系统，其他来源的直接拒绝。

## 🛠️ 四、 本项目实战学习计划

目前您的项目 `sentinel-and-seata` 还是一个空的基于 Java 17 的 Maven 工程。
为了让您能真正在代码中感受到 Sentinel 的威力，我建议采用 **Spring Cloud Alibaba** 体系来构建演练环境。

按照以下步骤，我将协助您逐步完成配置和代码验证：

### 第一阶段：项目骨架搭建
1. 在根 `pom.xml` 中引入 Spring Boot 和 Spring Cloud Alibaba 父依赖，管理版本统一。
2. 创建一个基于 Spring Boot 的子模块（例如 `order-service`）。

### 第二阶段：接入 Sentinel
1. **启动 Sentinel 控制台**：下载并运行 Sentinel Dashboard（一个轻量级的独立管控台服务）。
2. 在 `order-service` 中引入 Sentinel 依赖（`spring-cloud-starter-alibaba-sentinel`）。
3. 在 `application.yml` 中配置 Sentinel，使其连接到刚刚启动的控制台。

### 第三阶段：代码实战与控制台体验 (核心环节)
在这步中，我将为您编写测试接口，并带您验证：
- [ ] 编写一个简单的 `/hello` 接口，在控制台对其设定 **QPS 限流规则**。
- [ ] 结合 `@SentinelResource` 注解，针对限流后的请求，编写**自定义的回退逻辑 (兜底方法 fallback/blockHandler)**。
- [ ] 演示 **慢调用熔断**，主动使用 `Thread.sleep` 模拟慢响应，观察熔断器开环闭环。
- [ ] 演示 **热点参数限流**。

### 第四阶段：过渡到 Seata（后续）
理解了 Sentinel 防护机制后，我们再在这个项目中加入 `account-service` 和 `storage-service`，带您学习分布式事务（Seata）。
